GridRow (class in module ui.grid )

Description

GridRow is the viewModel for one logical row on the grid. A grid Row is not necessarily a one-to-one relation to gridOptions.data.

Usage

GridRow(entity, index, reference);

Parameters

  • entity – {object} –

    the array item from GridOptions.data

  • index – {number} –

    the current position of the row in the array

  • reference – {Grid} –

    to the parent grid

Methods

  • clearRowInvisible(row)

    Clears an override on the row that forces it to always be invisible. Emits the rowsVisibleChanged event if it changed the row visibility.

    This method can be called from the api, passing in the gridRow we want altered. It should really work by calling gridRow.clearRowInvisible, but that's not the way I coded it, and too late to change now. Changed to just call the internal function row.clearThisRowInvisible().

    Parameters

    • row – {GridRow} –

      the row we want to clear the invisible flag

  • clearRowInvisible(reason, fromRowsProcessor)

    Clears any override on the row visibility, returning it to normal visibility calculations. Emits the rowsVisibleChanged event

    Parameters

    • reason – {string} –

      the reason (usually the module) for the row to be invisible. E.g. grouping, user, filter

    • fromRowsProcessor – {boolean} –

      whether we were called from a rowsProcessor, passed through to evaluateRowVisibility

  • evaluateRowVisibility(fromRowProcessor)

    Determines whether the row should be visible based on invisibleReason, and if it changes the row visibility, then emits the rowsVisibleChanged event.

    Queues a grid refresh, but doesn't call it directly to avoid hitting lots of grid refreshes.

    Parameters

    • fromRowProcessor – {boolean} –

      if true, then it won't raise events or queue the refresh, the row processor does that already

  • getEntityQualifiedColField(col)

    returns the qualified field name minus the row path ie: entity.fieldA

    Parameters

    • col – {GridCol} –

      column instance

    Returns

    {string} –

    resulting name that can be evaluated against a row

  • getQualifiedColField(col)

    returns the qualified field name as it exists on scope ie: row.entity.fieldA

    Parameters

    • col – {GridCol} –

      column instance

    Returns

    {string} –

    resulting name that can be evaluated on scope

  • setRowInvisible(row)

    Sets an override on the row that forces it to always be invisible. Emits the rowsVisibleChanged event if it changed the row visibility.

    This method can be called from the api, passing in the gridRow we want altered. It should really work by calling gridRow.setRowInvisible, but that's not the way I coded it, and too late to change now. Changed to just call the internal function row.setThisRowInvisible().

    Parameters

    • row – {GridRow} –

      the row we want to set to invisible

  • setThisRowInvisible(reason, fromRowsProcessor)

    Sets an override on the row that forces it to always be invisible. Emits the rowsVisibleChanged event if it changed the row visibility

    Parameters

    • reason – {string} –

      the reason (usually the module) for the row to be invisible. E.g. grouping, user, filter

    • fromRowsProcessor – {boolean} –

      whether we were called from a rowsProcessor, passed through to evaluateRowVisibility

Properties

  • entity

    A reference to an item in gridOptions.data[]

  • grid

    A reference back to the grid

  • height

    height of each individual row. changing the height will flag all row renderContainers to recalculate their canvas height

  • uid

    UniqueId of row

  • visible

    If true, the row will be rendered
